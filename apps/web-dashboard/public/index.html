Why .HMAN?

How It Works

You're always in the middle

Every AI request goes through you. Not around you. Not past you. Through you.

1

Generate a session code

Message .HMAN on Signal with "code". You'll receive a 6-character code valid for 5 minutes.

2

Give the code to an AI

Tell Claude, GPT, or any AI your code. They connect to your session—nothing more.

3

Approve or deny each request

When they want your calendar, contacts, or anything else, we ask you first. Reply Y or N.

4

Revoke anytime

Send "revoke" and every session ends immediately. No waiting period. No fine print.

Technical Information

Specifications

Transparent architecture for developers and security researchers.

Session Codes

6-character alphanumeric codes (A-Z, 2-9, excluding ambiguous characters I, O, 0, 1). Cryptographically random. Single-use. Expire after 5 minutes or first use, whichever comes first.

Signal Protocol

All user communication uses Signal's end-to-end encryption. We integrate via signal-cli as an unofficial client. Messages are encrypted using the Signal Protocol (X3DH key agreement + Double Ratchet).

MCP Integration

AI assistants connect via Model Context Protocol (MCP). Our server mediates all resource requests, routing each to the user for approval before any data is shared.

Zero Data Retention

We don't store your data. Session codes, approval logs, and phone numbers exist only in memory during active sessions. Nothing is written to disk. Nothing persists.

Audit Logging

Every request and response is logged—but only on your end. You can see what was requested, when, and how you responded. We keep nothing.
